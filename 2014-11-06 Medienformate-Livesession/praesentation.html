HTML5, Flash oder Beides?

<video>-Element
Vorteile
Nachteile
  • einfacher Code
  • keine Plug-ins nötig
  • schnellere Ladezeiten
  • kann mit JavaScript gesteuert werden
  • kein Streaming
  • kein stylen mit CSS
  • wenige Metadaten (SEA?)
  • Fullscreen -> Browser überlassen
<video>-Element
<video ...
width="" Breite des Videos
height="" Höhe des Videos
poster="" Bild, wenn Video noch nicht gestartet
autoplay Video wird sofort gestartet
controls play, pause, stop...
autobuffer Speichert das Video beim Laden zwischen
muted Video auf stumm geschaltet
loop Nach Ende beginnt es wieder von Vorne
Code-Beispiel

   <video controls muted width="500" height="300" poster="chameleon.jpg" autobuffer>

         <source src="wildlife.mp4" type="video/mp4">

         <source src="wildlife.m4v" type="video/mp4">

         <source src="wildlife.ogv" type="video/ogg">

   </video>

<audio>-Element
<audio ...
autoplay Audio wird sofort gestartet
muted Audio zu beginn stumm
controls Leiste mit Steuerungsfunktionen (z.B. play)
loop Audio in Dauerschleife
volume Lautstärke (0.0 - 1.0)
Code-Beispiel

   <audio controls muted loop>

         <source src="audio-bsp.mp3" type="audio/mp3">

         <source src="audio-bsp.wav" type="audio/wav">

         <source src="audio-bsp.ogg" type="audio/ogg">

   </audio>

<picture>-Element

Warum nicht img?
Vorteile von <picture>
  • Bisheriges Verfahren: Einbindung durch <img> und Viewport-Anpassung durch CSS
  • Nachteil: Auch bei mobilen Geräten Einbindung hochauflösender Bilder (unnötig lange Ladezeiten)

  • <picture> ermöglicht alternative Bildquellen für verschiedene Viewports
    -->
    Ideal für responsives Design!
  • Bei fehlender Browserunterstützung: Fallback-Funktion (Angabe des üblichen img-Bildes)
  • Nachteil: Bislang lediglich von Chrome Chanary (Dev. Version) unterstützt!

Live-Übung

Erstellen Sie eine Seite und binden Sie auf dieser eine Videodatei und eine Audiodatei ein.

(Browserkompatibilität muss gewährleistet sein!)

Das Audiomaterial soll dauerhaft im Hintergrund laufen und nur bei Start des Videos pausieren.

Das <video>-Element darf nicht das Attribut controls verwenden, stattdessen sollen mit JavaScript Buttons erstellt werden für folgende Funktionen:

  • Play/Pause/Stop
  • Vorspulen/Zurückspulen
  • Lautstärkeregelung
