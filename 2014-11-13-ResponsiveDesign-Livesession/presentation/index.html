Der Begriff „Responsive Webdesign“
wurde erstmals von Ethan Marcotte
im Mai 2010 in einem Artikel
für das Magazin „A List Apart“ verwendet.


„But what’s next? An iPad website?
An N90 website? Can we really continue to commit
to supporting each new user agent
with its own bespoke experience?
At some point, this starts to feel like a zero sum game.
But how can we—and our designs—adapt?“

Geschichte

Warum das Ganze?


adaptive vs. responsive

Es wird zwischen adaptiven (anpassungsfähigen, „adaptive“)
und reaktionsfähigen („responsive“) Layouts unterschieden.

adaptive layout

  • Für verschiedene Displaygrößen optimierte Website
  • Meistens 3 verschiedene Displaygrößen
    (Desktop, Tablet und Smartphone)
  • Je ein Layout für die 3 Ansichten, also ein starres Gestaltungsraster

Vorteile adaptive

  • Leichtere Planung und Arbeit mit Mockups, Wireframe und Skizzen, da feste Abmessungen existieren
  • Viel gestalterischer Freiraum, da mit einem starren Raster gearbeitet wird
  • Technisch recht unkomplizierte und zeitsparende Umzusetzung

Nachteile adaptive

  • Nur für bestimmte Displaygrößen optimiert
  • Diese müssen erst durch eine Zielgruppenanalyse bestimmt werden
  • Häufige Fehldarstellung bei unpassenden Displaygrößen

Responsive layout

  • Für jede erdenkliche Displaygröße optimierte Website
  • Flüssiges Gestaltungsraster
  • Dynamisches Layout und perfekt angepasster Inhalt

Vorteile responsive

  • Zukünftige Geräte werden automatisch mit abgedeckt
  • Die Informationen und der Inhalt werden bestmöglichst dargestellt
  • optimale Anpassung der Website an jede Bildschirmauflösung und -größe

Nachteile responsive

  • Komplizierter bei der Planung, Gestaltung sowie der technischen Umsetzung
  • Aufwendiges Einfügen der Inhalte
  • Dadurch höhere Kosten

Worauf ist zu achten?

  • Kein festes Raster
  • Keine festen Schriftgrößen
  • Keine festen Bildgrößen
  • Prozentuale statt fest vorgegebene Pixelwerte
  • Mobile first
  • Website als ganzes betrachten

Relative Wertangaben

"px" "Pixel"
% Prozent: Je nach CSS-Eigenschaft relativ zur elementeigenen Größe, oder zu der des Elternelements, oder zu einem allgemeinen Kontext
em Bezogen auf die Schriftgröße des Elements. Ausnahme: Wenn die font-size-Eigenschaft (also die Schriftgröße) selbst mit dieser Maßangabe gesetzt wird, bezieht sich die Angabe auf die Schriftgröße des Elternelements.
rem (root em): Bezieht sich auf die Schriftgröße des Wurzelelements.

    Quelle: Maßeinheiten im Web (fh-koeln.github.io/wba1-2014)

Beispiel: Medienabfrage in der CSS-Ressource 
/* CSS-Regeln für alle Ausgabegeräte */
@media print {
  /* zusätzliche oder abweichende CSS-Regeln für Drucker */
}
Empfehlung: Aus Gründen der Performance sollten Sie diese Variante der Einbindung wählen.

Einbindung - Media Queries


<link rel="stylesheet" href="kompakt.css" media="(orientation: portrait)">
Das Stylesheet kompakt.css wird eingebunden, wenn die Inhalte auf Seiten im Hochformat ausgegeben werden. 

<style type="text/css" media="(color)">
/* Farbangaben. */
</style>
Die im style-Element angegebenen Farbangaben werden verarbeitet, wenn das Ausgabemedium Farben darstellen kann. 

@import 'layout.css' (min-width: 50em);
Das Stylesheet layout.css kommt zur Anwendung, wenn der Anzeigebereich des Ausgabemediums mindestens 50em beträgt. Die Einheit em bezieht sich dabei auf die Standardschriftgröße des Browsers.

Gruppierung und Kombination
- Media Queries

Mithilfe von Kommata können Medienabfragen gruppiert werden.
Sobald mindestens eine der Abfragen zutrifft, werden die Deklarationen angewendet. 
@media print, embossed { /* Formate für Printmedien. */ }

Mehrere Medienmerkmale können mit dem Schlüsselwort and (und) verbunden werden.
Ein Stylesheet wird nur dann verarbeitet, wenn alle mit "and" verbundenen Kriterien erfüllt werden 
@media (min-width: 30em) and (max-width: 60em) { /* Kompaktes Layout */ }
@media print and (color), screen and (color) { /* Farbangaben */ }

Medientypen - Media Queries

all Alle Ausgabemedien. Dies ist der Standardwert.
print für Drucker, die Inhalte sichtbar auf Papier drucken.
screen (Computer-)Bildschirme
speech für Sprachsynthesizer gedacht.
Die Angabe ist für ein zukünftiges CSS-Modul reserviert und wird bisher nicht verwendet, soll aber den Medientyp „aural“ ersetzen.

Medienmerkmale - Media Queries

  • width (Breite)
    • @media (width: 60em) {/*Breite entspricht genau 60em*/}
@media (min-width: 50em) {/*Breite beträgt mindestens 50em*/}
@media (max-width: 70em) {/*Breite beträgt höchstens 70em*/}

  • height (Höhe)
  • device-width (Geräte-Breite)
  • device-height (Geräte-Höhe)
  • orientation (Orientierung):
    landscape (Querformat) oder portrait (Hochformat)

  • aspect-ratio (Seitenverhältnis)
    • @media (aspect-ratio: 4/3) {/*Fall1*/}
@media (min-aspect-ratio: 4/3) {/*Fall2*/}
@media (max-aspect-ratio: 4/3) {/*Fall3*/}

Einfaches Beispiel zu Media Queries

<style>
@media screen and (min-width: 960px) {
	#hello {

@media screen and (max-width: 960px) {

@media screen and (max-width: 320px) {

Übung

    01. Setzen Sie den Viewport-Meta-Tag an die richtige Stelle.

    02. Öffnen Sie die CSS-Datei "layout.css", die sich im Ordner "styles" befindet, um sie zu bearbeiten.

    03. Ändern Sie den Pixel-Wert der body-Breite in einen prozentualen Wert. Entscheiden Sie sinnvoll.

    04. Schreiben Sie zusätzlich zur body-Breite eine Mindestbreite und eine Maximalbreite (320px & 1600px)

    05. Gibt es einen weiteren Pixel-Wert? Ändern Sie diesen sinnvoll.

    06. Was stimmt mit dem Elefanten nicht? Ordnen Sie ihm eine prozentuale Maximalbreite zu. [Tipp: Der Elefant hat die Klasse .image_right]

Übung

    07. Erstellen Sie eine neue CSS-Datei mit dem Namen "media_queries.css".

    08. Binden Sie die erstellte Datei im "head" der HTML-Datei ein.

    09. Erstellen Sie zwei Media Queries (Medienabfragen) mit folgenden Maximalbreiten: 1024px & 800px

    10. Setzen Sie die Breite des "content" und der "sidebar" auf 100% beim dazu passenden Media Query.

    11. Setzen Sie den Textumfluss (float) auf die Normaleinstellung beim gleichen Media Query wie bei Nr.10.

    12. Verändern Sie bei dem anderen Media Query sinnvoll folgendes: #header & #header h1

Übung

    13. Was passiert, wenn Sie folgendes in die CSS-Datei "media_queries.css" kopieren?

    #header {
transition: height 0.3s linear;
}
#header h1 {
transition: font-size 0.3s linear;
}

    14. Wenn Sie noch Lust und Zeit haben, können Sie weitere Media Queries erzeugen oder in bestehende Media Queries Sachen ergänzen, die Sie für sinnvoll halten.
