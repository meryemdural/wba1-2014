Aufgabenstellung


Projekt: Medieninformatik-Website der FH Köln barrierefrei gestalten
Zielgruppe: Menschen mit Sehbehinderungen

Barrierefreiheit im Internet

für das Projekt relevante Richtlinen (WCAG)


  • Skalierbarkeit der Schrift
  • starke Kontraste
  • Informationen nicht allein über Farben vermitteln
  • Navigation allein über die Tastatur muss möglich sein
  • Flash-Objekte vermeiden
  • klar strukturierter Text
  • Textalternativen bei Grafiken verwenden
  • leicht zu verstehende Navigation

Umsetzung

  • Erarbeitung von Personas -> User Needs
  • Recherchen zu barrierefreien Websiten
  • Festlegung auf zwei Unterseiten
    • Über den Medieninformatik Bachelorstudiengang
    • Modulübersicht und -beschreibungen
  • Analyse der ausgewählten Unterseiten
  • Herausarbeitung der zu treffenden Änderungen
  • Entscheidung Code neu zu schreiben
  • Entwicklung eines Konzeptes
    • Kontrastansicht
    • vergrößerte Ansicht

Problematik

  • Farbschema der MI beibehalten?
  • Framework verwenden?
  • Umsetzung mit Javascript effektiv?
  • Kontrastansicht und vergrößerte Ansicht notwendig?
