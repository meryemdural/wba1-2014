Die Erstellung benutzerdefinierter HTML-Templates

Möchtest du deinem Blog einen individuellen Look verpassen? Wenn du mit HTML umgehen kannst, bist du hier genau richtig! Wenn nicht, kannst du in unserem Template-Garten Hunderte von schicken Templates finden.

Einführung

Im Zentrum von Tumblr stehen sieben Eintragstypen: Text, Bilder, Links, Zitate, Chats, Audio und Video. Dein Template sollte unbedingt an diese Eintragstypen angepasst sein, damit die Einträge überall gut aussehen: in deinem eigenen Blog, in den Dashboards deiner Follower und als gerebloggte Einträge in anderen Blogs.

Tipps

Versuche bitte in Texteinträgen auf Bilder aus externen Quellen und eingebettete Medien zu verzichten. Das kommt auf dem Dashboard nicht so toll rüber und kann Probleme bei benutzerdefinierten Templates verursachen. Erstelle lieber Videoeinträge für Videos, Bildeinträge für Bilder etc.

Leg los

  1. Klicke auf den Namen des Blogs oben im Dashboard oder in der Blogliste oben rechts.
  2. Klicke in der rechten Spalte auf "Anpassen".
  3. Klicke auf "HTML bearbeiten" links unter dem Miniaturbild des Templates.

Tumblr bietet zwei Typen von speziellen Operatoren, die Inhalte in deiner HTML rendern.

Variablen werden verwendet, um dynamische Daten wie den Titel oder die Beschreibung des Blogs anzugeben:

Beispiel

<html>
    <head>
        <title>{Title}</title>
    </head>
    <body>
        ...
    </body>
</html>

Blocksätze werden entweder dazu verwendet, einen HTML-Blocksatz für einen Datensatz (z. B. deine Einträge) zu rendern oder einen HTML-Blocksatz bei Bedarf zu rendern (z. B. einen "Vorige Seite"-Link).

<html>
    <body>
        <ol id="posts">{block:Posts}
                <li> ... </li>{/block:Posts}
        </ol>
    </body>
</html>

Hier ist ein Beispiel für das komplette Markup eines Templates:

<html>
    <head>
        <title>{Title}</title>
        <link rel="shortcut icon" href="{Favicon}">
        <link rel="alternate" type="application/rss+xml" href="{RSS}">
        {block:Description}
            <meta name="description" content="{MetaDescription}" />
        {/block:Description}
    </head>
    <body>
        <h1>{Title}</h1>

        {block:Description}
            <p id="description">{Description}</p>
        {/block:Description}

        <ol id="posts">
            {block:Posts}{block:Text}
                    <li class="post text">
                        {block:Title}
                            <h3><a href="{Permalink}">{Title}</a></h3>
                        {/block:Title}{Body}
                    </li>
                {/block:Text}{block:Photo}
                    <li class="post photo">
                        <img src="{PhotoURL-500}" alt="{PhotoAlt}"/>

                        {block:Caption}
                            <div class="caption">{Caption}</div>
                        {/block:Caption}
                    </li>
                {/block:Photo}{block:Panorama}
                    <li class="post panorama">
                        {LinkOpenTag}
                            <img src="{PhotoURL-Panorama}" alt="{PhotoAlt}"/>
                        {LinkCloseTag}{block:Caption}
                            <div class="caption">{Caption}</div>
                        {/block:Caption}
                    </li>
                {/block:Panorama}{block:Photoset}
                    <li class="post photoset">
                        {Photoset-500}{block:Caption}
                            <div class="caption">{Caption}</div>
                        {/block:Caption}
                    </li>
                {/block:Photoset}{block:Quote}
                    <li class="post quote">
                        "{Quote}"

                        {block:Source}
                            <div class="source">{Source}</div>
                        {/block:Source}
                    </li>
                {/block:Quote}{block:Link}
                    <li class="post link">
                        <a href="{URL}" class="link" {Target}>{Name}</a>

                        {block:Description}
                            <div class="description">{Description}</div>
                        {/block:Description}
                    </li>
                {/block:Link}{block:Chat}
                    <li class="post chat">
                        {block:Title}
                            <h3><a href="{Permalink}">{Title}</a></h3>
                        {/block:Title}

                        <ul class="chat">
                            {block:Lines}
                                <li class="{Alt} user_{UserNumber}">
                                    {block:Label}
                                        <span class="label">{Label}</span>
                                    {/block:Label}{Line}
                                </li>
                            {/block:Lines}
                        </ul>
                    </li>
                {/block:Chat}{block:Video}
                    <li class="post video">
                        {Video-500}{block:Caption}
                            <div class="caption">{Caption}</div>
                        {/block:Caption}
                    </li>
                {/block:Video}{block:Audio}
                    <li class="post audio">
                        {AudioEmbed}{block:Caption}
                            <div class="caption">{Caption}</div>
                        {/block:Caption}
                    </li>
                {/block:Audio}{/block:Posts}
        </ol>

        <p id="footer">
            {block:PreviousPage}
                <a href="{PreviousPage}">&#171; Previous</a>
            {/block:PreviousPage}{block:NextPage}
                <a href="{NextPage}">Next &#187;</a>
            {/block:NextPage}

            <a href="/archive">Archive</a>
        </p>
    </body>
</html>

Template-Optionen

Benutzerdefinierte Farben aktivieren

Wenn du spezielle Metacolor-Tags in deinem Template unterbringst, können User diese Farben ganz einfach auf der Anpassen-Seite ändern.

Du kannst auch {BackgroundColor}, {TitleColor} oder {AccentColor} als Standardeinstellungen festlegen, die die Werte des globalen Erscheinungsbildes übernehmen, sofern der User nicht manuell eine andere Farbe einstellt.

Beispiel

<html>
    <head>
        <!-- DEFAULT COLORS -->
        <meta name="color:Background" content="#eee"/>
        <meta name="color:Content Background" content="#fff"/>
        <meta name="color:Heading text" content="{TitleColor}"/>
        <meta name="color:Text" content="#000"/>

        <style type="text/css">
            #title {
                color: {color:Heading text};
            }
            #content {
                background-color: {color:Content Background};
                color: {color:Text};
            }
        </style>
    </head>
    <body bgcolor="{color:Background}">
        <h1 id="title">Title</h1>
        <div id="content">
            ...
        </div>
    </body>
</html>

Benutzerdefinierte Bilder aktivieren

Wenn du spezielle Meta-Bild-Tags in deinem Template unterbringst, können User benutzerdefinierte Bilder verwenden, ohne dein Template zu bearbeiten. Bild-Variablen (z. B. {image:Logo} ) werden ein durchsichtiges, 1 Pixel großes Quadrat liefern, wenn kein Bild eingestellt ist.

Du kannst auch {HeaderImage} als Standardeinstellung festlegen, die die Werte des "Header-Bildes" im globalen Erscheinungsbild übernimmt, sofern der User nicht manuell ein anderes Bild einstellt.

</html>

Falls du in deinen Tumblr-Einstellungen die Twitter-Integration aktiviert hast, kannst du die Javascript-Datei /tweets.js in deinem Hauptblog unterbringen, um deine letzten Tweets anzeigen zu lassen. Diese Datei führt die Callback-Funktion recent_tweets() aus. Dabei werden die JSON-Daten der Twitter-API als einzige Parameter verschickt.

VariableBeschreibung
{block:Twitter} {/block:Twitter}Wird gerendert, wenn du die Twitter-Integration aktiviert hast.

    <script type="text/javascript">
        function recent_tweets(data) {
            for (i=0; i<data.length; i++) {
                document.getElementById("tweets").innerHTML =
                    document.getElementById("tweets").innerHTML +
                    '<a href="http://twitter.com/{TwitterUsername}/status/' +
                    (data[i].id_str ? data[i].id_str : data[i].id) +
                    '"><div class="content">' + data[i].text +

Umwandlung von Variablen

Wenn man Variablen bestimmte umwandelnde Begriffe voranstellt, wird Tumblr diese Variablen in speziellen Formaten ausgeben. Das ist nützlich, wenn man Daten an Javascript weiterleitet etc. Tumblr unterstützt aktuell fünf Umwandlungen:

VariableBeschreibung
PlaintextPlaintext vor einer beliebigen Template-Variablen führt zur Ausgabe des Strings, wobei HTML-Tags entfernt und die entsprechenden Zeichen zu HTML-Entities umgewandelt werden, damit man sie bedenkenlos in HTML-Attributen etc. unterbringen kann.
JavascriptJS vor einer beliebigen Template-Variablen führt zur Ausgabe eines Javascript-Strings (in Anführunsgzeichen).
Javascript PlaintextJSPlaintext vor einer beliebigen Template-Variablen führt zur Ausgabe eines Javascript-Strings (in Anführunsgzeichen), wobei HTML-Tags entfernt und die entsprechenden Zeichen zu HTML-Entities umgewandelt werden.
URLEncodedURLEncoded vor einer beliebigen Template-Variablen führt zur Ausgabe eines URL-kodierten Strings.
RGBRGB vor einer Farbvariablen wandelt die Hex-Ausgabe in eine RGB-Ausgabe um.

<a href="{URL}" title="{PlaintextName}">{Name}</a>
